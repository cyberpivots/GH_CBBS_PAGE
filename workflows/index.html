Nearby room-client onboarding

A nearby user joins a room node Wi-Fi surface, opens the browser UI, applies or logs in, and browses approved rooms.

  1. Operator confirms the room node is serving its local WPA2 SoftAP.
  2. Nearby user connects a phone, tablet, or laptop to the room SSID.
  3. Browser opens the room-node HTML interface.
  4. User applies, logs in, or uses an approved account path.
  5. Approved rooms and policy views become available according to access level.

The flow depends on local proximity and available AP client capacity.
